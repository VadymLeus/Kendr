# 🚀 Kendr: NoCode платформа конструювання вебресурсів

**Kendr** — це сучасна self-hosted NoCode платформа, яка дозволяє користувачам без навичок програмування створювати повноцінні вебресурси та інтернет-магазини.

На відміну від закритих конструкторів (SaaS), система базується на компонентному підході із серіалізацією структури сторінок у JSON, що забезпечує гнучкість, відсутність Vendor Lock-in та повний контроль над власними даними.

## ✨ Ключові Можливості

* 🎨 **Візуальний Drag-and-Drop редактор:** Інтуїтивне конструювання сторінок з готових блоків (Hero, Features, Text тощо), автоматичне збереження та керування глобальними компонентами (Header, Footer).
* 🛒 **Повноцінний E-commerce модуль:** Створення товарів, категорій, налаштування мультивалютності. Інтегрований кошик та система оформлення замовлень з антиспам-лімітами.
* ⚙️ **Гнучке управління сайтом:** Динамічне приховування модулів (наприклад, вимкнення магазину), налаштування Cookie-банерів для відповідності юридичним нормам.
* 🛡️ **Модерація та Безпека:** Спеціалізована Адмін-панель для контролю платформи. Можливість блокування сайтів та акаунтів користувачів із фіксацією причин (`ban_reason`).
* 🧪 **Забезпечення якості:** Критичний функціонал покритий багаторівневим тестуванням, включаючи E2E-сценарії повного циклу користувача.

## 🛠️ Технологічний Стек

### Backend (API & Серверна логіка)
* **Node.js та Express:** Швидка та масштабована серверна частина.
* **MySQL:** Реляційна база даних (гібридна модель зберігання: реляційні зв'язки + JSON для конфігурації сайтів).
* **JWT (JSON Web Tokens):** Захист маршрутів та безпечна автентифікація.
* **Multer & Sharp:** Завантаження та оптимізація медіафайлів (зображення товарів, аватари).

### Frontend (Клієнтська частина)
* **React:** Компонентна архітектура користувацького інтерфейсу.
* **React Router:** Клієнтський роутинг.
* **Axios:** Взаємодія з REST API.
* **React Context:** Глобальне управління станами (сесії, кошик, налаштування).

### Тестування
* **Playwright / Vitest:** Автоматизоване E2E та інтеграційне тестування користувацьких сценаріїв.

## ⚙️ Встановлення та Запуск (Self-hosted)

### Вимоги
* [Node.js](https://nodejs.org/) (v16+)
* [MySQL](https://www.mysql.com/)

### 1. Налаштування Backend

**Клонуйте репозиторій:**
```bash
git clone [https://github.com/VadymLeus/Kendr.git](https://github.com/VadymLeus/Kendr.git)
cd Kendr
```

**Встановіть залежності сервера:**
```bash
cd backend
npm install
```

**Ініціалізація Бази Даних:**
* Запустіть ваш MySQL сервер.
* Виконайте SQL-скрипт `database_schema.sql`, щоб створити базу даних `KendrDB` та структуру таблиць.

**Конфігурація середовища:**
Створіть файл `.env` у директорії `backend` та вкажіть ваші налаштування:
```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_db_password
DB_NAME=KendrDB
JWT_SECRET=your_super_secret_jwt_key
PORT=5000
```

**Запуск сервера:**
```bash
npm run dev
```
API буде доступне за адресою http://localhost:5000.

### 2. Налаштування Frontend

**Відкрийте новий термінал, перейдіть у корінь проєкту та встановіть залежності клієнта:**
```bash
cd frontend
npm install
```

**Запустіть клієнтську частину:**
```bash
npm run dev
```
Платформа відкриється у вашому браузері за адресою http://localhost:5173.
